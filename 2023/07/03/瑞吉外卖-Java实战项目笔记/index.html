导入数据表

  • 数据表说明

  • 通过可视化图形界面导入数据表

    1. 右击需要添加数据表的数据库，点击 执行SQL文件

    2. 点击开始按钮，执行SQL文件，执行结束后出现successfully表示导入成功

  • 通过命令行终端导入

    1. 通过 source xxxSQL文件路径 命令即可导入

    2. 使用命令导入需要注意！如果SQL文件路径中包含中文则会导入失败，并且直接导入可能会出现乱码的情况~

Maven项目搭建

创建Maven项目

  1. 通过IDEA创建一个Maven项目。创建完成后，注意检查项目编码、maven仓库配置、jdk配置等。

  2. 导入pom.xml

  3. 创建包并编写resource/application.yml配置

  4. 编写项目的主程序 src/main/java/com/zsycx/reggie/ReggieTakeOutApplication.java

  5. 启动SpringBoot项目

  6. 导入前端页面静态文件到src/main/resources

    导入完成此时是不能够直接访问的，因为SpringBoot默认只能访问src/main/resources/static中的静态资源

    需要在项目中添加配置类进行静态资源放行，创建src/main/java/com/zsycx/reggie/WebMVCConfig.java

    配置完成之后我们发现就可以正常访问啦~

代码开发

完善项目结构

  1. 创建对应的 controller、service、mapper、domain 包

  2. 创建实体类 com/zsycx/reggie/domain/Employee.java

    • @Data 该注解由lombok提供，是以下注解的集合 @Setter @Getter @ToString @EqualsAndHashCode @RequiredArgsConstructor（必要构造方法）方法
    • @TableField 该注解由Mybatis-Plus提供，字段注解（非主键），对标记实体类中的字段进行处理
      • fill属性 使用字段填充策略

  3. 创建对应的Mapper和Service

    创建Mapper类 com/zsycx/reggie/mapper/EmployeeMapper.java

    需要继承 BaseMapper<对应实体类> 接口

    创建Service接口 com/zsycx/reggie/service/EmployeeService.java

    需要继承 IService<对应实体类> 接口

    创建Service接口实现类 com/zsycx/reggie/service/impl/EmployeeServiceImpl.java

    需要继承 ServiceImpl<对应实体Mapper类, 对应实体类> 并且实现 EmployeeService 接口

  4. 创建控制器类 com/zsycx/reggie/controller/EmployeeController.java

    • @Slf4j lombok提供的日志注解
    • @RestController 这个注解是@ResponseBody和@Controller的结合注解，@ResponseBody的作用是，将方法的返回值转换为Json响应给客户端
    • @RrequestMapping 设置映射地址，应用在类上表示设置为该类中所有方法映射地址的父地址
    • @Autowired 自动装配

  5. 创建common包，并导入通用类 Result

  6. 完善后的项目结构图

登录逻辑处理流程图

在Controller中编写login方法映射

  • @RequestBody：该注解作用是将请求到的Json对象转换为Java对象
  • HttpServletRequest：request Http请求对象，用于将用户ID存入Session中，以便使用
  • DigestUtils：Apache Commons Codec 库提供了 DigestUtils 工具类，它提供了一种方便的方法来计算数据哈希值。该类提供了多种哈希算法，如MD5、SHA-1、SHA-256等。

登录测试

目前数据库中只有一条employee数据，username:admin password:123456，但是密码是经过MD5加密过的

访问 http://localhost:8080/backend/page/login/login.html 进行测试

可以通过编辑器的Debug来测试，使用Debug前建议修以下前端的请求超时时间src/main/resources/backend/js/request.js文件中的timeout属性，这样就不会出现接口超时的情况

点击登录以后能够正常跳转到/backend/index.html，并且能够在调试工具中查看到后端响应给前端的数据，表示登录成功~

如果不能够进行跳转那么表示登录失败，具体失败原因可以根据提示进行判断

退出登录

点击右上角退出登录按钮后请求的地址请求地址： http://localhost:8080/employee/logout

在Controller中编写logout方法映射

退出登录成功以后前端也会将登录时存储的用户信息移除
